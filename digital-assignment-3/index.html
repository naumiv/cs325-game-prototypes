Dig!

  • Originality: My game takes the concept of a paleontological dig site and turns it into a chance-based game. This is different from other games that may have the same premise but make into a more puzzle, bone-assembly game or adventure game like Jurassic Park: The Game.
  • Technical Merit: I included the feel of a day to night cycle by animating the background and creating custom gradients for the sky. I was able to incorporate background music as well as action based sound effects. I also included a variety of both negative and positive "events" that could happen to the player, which would effect their inventory and rank in real time.
  • Prototype Postmortem: I learned how to work with data objects in Phaser and link it to a specific image, which allowed me to treat the images like objects with fields i.e. money. This was my first time working with audio as well, so I learned how to play it in the background and only play when linked to a specific action. Similarly, this was my first time using mouse / point controls, so I now better understand the Javascript and function calls related to it. A major part of the game was updating the values after the player clicked, so I needed to learn to handle those updates with Javascript functions, which took a while. Finally, I also implemented a animated background which was a little different from creating a player-controlled sprite, so I now better understand the differences between the two. The background used a timer so I read the documentation and learned more about the timing abilities of phaser.
